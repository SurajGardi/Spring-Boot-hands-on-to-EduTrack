EduTrack Student Portal – Full Stack Backend with ResponseEntity & Status Codes
------------------------------------------------------------------------------

Project Name:
-------------
8_EduTrack_Student_Portal_FullStack_Complete_EntityResponce

Objective:
----------
This project improves the previous EduTrack backend by adding:
✔ ResponseEntity
✔ HTTP Status Codes
✔ Improved API responses
✔ Client-friendly communication
✔ Minimal exception handling

These adjustments prepare the backend for real UI/front-end usage.

------------------------------------------------------------------------------

1. What Changed in This Version?
--------------------------------

Compared to previous version:
7_EduTrack_Student_Portal_FullStack_Complete

NEW improvements include:

(1) Usage of ResponseEntity
---------------------------
Old:
return object

New:
return new ResponseEntity<>(object, HttpStatus.OK)

In Controller: :contentReference[oaicite:0]{index=0}

Benefits:
---------
✔ Full control over HTTP responses
✔ Works better with REST clients (React, Angular, Vue, Postman)
✔ Allows validation & error handling

(2) HTTP Status Code Handling
-----------------------------

Added Status Codes:
✔ 200 OK
✔ 201 CREATED
✔ 204 NO_CONTENT
✔ 400 BAD_REQUEST
✔ 404 NOT_FOUND

Each corresponds to a valid REST action.

(3) Null and Empty Handling
---------------------------

Example in getAll():
---------------------
if list empty → return NOT_FOUND (404)
if list present → return OK (200)

(4) Exception Safety (Basic)
----------------------------

POST now wrapped in try-catch:
-------------------------------
try → CREATED (201)
catch → BAD_REQUEST (400)

------------------------------------------------------------------------------

2. Status Code Reference Table
------------------------------

OPERATION         STATUS CODE     MEANING
-----------------------------------------------------
GET /batches       200 OK         Data Found
                   404 NOT_FOUND  No Data Found

POST /batches      201 CREATED    Insert Successful
                   400 BAD_REQ    Invalid Input

GET /id/{id}       200 OK         Record Found
                   404 NOT_FOUND  No Record for ID

DELETE /id/{id}    204 NO_CONTENT Delete Successful

PUT /id/{id}       200 OK         Update Successful
                   404 NOT_FOUND  No Record to Update

------------------------------------------------------------------------------

3. Controller Behavior Breakdown
--------------------------------

(1) GET All
-----------
If data exists → returns list + 200 OK
If no data → returns 404 NOT_FOUND
:contentReference[oaicite:1]{index=1}

(2) POST Create Entry
----------------------
On success:
return CREATED + inserted object

On failure:
return BAD_REQUEST

(3) GET by ID
--------------
If present → 200 OK
If missing → 404 NOT_FOUND

(4) DELETE by ID
----------------
Always responds:
204 NO_CONTENT

(5) UPDATE by ID
----------------
If old exists:
✔ update fields
✔ return updated object + 200 OK
Else:
✔ return 404 NOT_FOUND
